## Summary

Implemented `EventBroadcaster` as a newtype wrapper around `tokio::sync::broadcast::Sender<WsEvent>` in `crates/anvilml-ipc/src/ws/broadcaster.rs`. The struct provides `new()`, `publish()`, and `subscribe()` methods for fanning out WebSocket events to subscribers. Added `tokio` dependency with `sync` feature and `uuid` dev-dependency for tests. Created 4 integration tests covering zero-subscriber, single-subscriber, multi-subscriber, and receiver validity scenarios. Added `Default` impl required by clippy's `new_without_default` lint.

## Deviations from Plan

- **`Default` impl added**: Clippy's `new_without_default` lint (triggered by `-D warnings`) requires a `Default` implementation when a `new()` method exists without one. Added `impl Default for EventBroadcaster { fn default() -> Self { Self::new() } }`. This was not in the original plan's Public API Surface but is a minimal, correct fix.
- **`uuid` dev-dependency added**: The plan did not specify a dev-dependency for `uuid`, but the tests construct `WsEvent` variants with `Uuid::new_v4()` fields. Added `uuid = { version = "1.23.4", features = ["v4"] }` to `[dev-dependencies]`.
- **`test_subscribe_returns_valid_receiver` assertion corrected**: The original plan's test name is `test_subscribe_returns_receiver`, but the actual test in the plan's Tests table uses `test_subscribe_returns_valid_receiver`. The test logic was adjusted: instead of asserting `recv()` doesn't return `RecvError::Closed` (which would hang indefinitely), the test uses `tokio::time::timeout` to confirm the receiver is still open (timeout elapsing proves the channel is not closed).
