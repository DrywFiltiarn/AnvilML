## Summary

Added unconditional `std::env::remove_var` teardown to four serial tests in `crates/anvilml-worker/src/managed.rs` (`handshake_completes_once`, `spawn_reaches_idle`, `spawn_ping_pong`, and `status_transitions`) to prevent environment variable leaks between serially-executed tests. Discovered that the `serial_test::serial` crate orders tests alphabetically, so `handshake_completes_once` runs before `spawn_ping_pong` and `status_transitions`. The latter two tests required `ANVILML_WORKER_MOCK=1` but didn't set it themselves, causing failures when `handshake_completes_once`'s teardown removed the variable. Added env-var setup to those two tests as a necessary deviation from the plan.

## Deviations from Plan

- **Added env-var setup to `spawn_ping_pong` and `status_transitions`**: The approved plan explicitly excluded changes to these two tests. However, during implementation I discovered that the `serial_test::serial` crate orders tests alphabetically (not by declaration order), so `handshake_completes_once` runs before `spawn_ping_pong` and `status_transitions`. The teardown in `handshake_completes_once` removes `ANVILML_WORKER_MOCK`, causing those two tests to fail because they depend on the variable being set. Adding `std::env::set_var("ANVILML_WORKER_MOCK", "1")` at the start of each test and unconditional teardown at the end makes them properly self-contained and prevents the same leak in either direction. This is a necessary deviation to achieve the plan's acceptance criteria (all tests passing).
- **Version bump**: The plan stated "No crate version bumps" under Out of Scope, but `FORGE_AGENT_RULES.md §12` mandates patch version bumps for any crate whose source files are modified. Bumped `anvilml-worker` from `0.1.9` to `0.1.10`.
