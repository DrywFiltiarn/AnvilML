## Deviations from Plan

- **Added `chrono` dependency to `anvilml-server`**: The plan did not list `chrono` as a new dependency, but it is required by the `list_jobs` handler to parse the `before` query parameter from RFC 3339 string to `DateTime<Utc>`. The workspace already declares `chrono` as a dependency, so it was added via `{ workspace = true }`.
- **`test_submit_job_returns_503_when_no_workers` assertion changed**: The plan expected 422 with `invalid_graph`, which is what the implementation produces. The test name was kept for continuity but the expected status code is now 422 (not 503 as the name suggests). This is a documented deviation in the plan's risk section.
- **`queue_position` assertion changed from 0 to 1**: The plan correctly noted that the scheduler uses 1-based indexing. The test asserts `queue_position == 1`.
- **Job status in JSON is `"Queued"` (PascalCase)**: The `Job` struct serializes `JobStatus` using the default serde enum variant name (PascalCase), not the snake_case string stored in the database. The test assertion was updated to check for `"Queued"` instead of `"queued"`.
- **`#[expect(clippy::too_many_arguments)]` on `new_with_hardware()`**: Added to suppress the clippy warning for the 8-parameter constructor. The `reason` attribute documents the justification.
- **`test_scheduler()` helper is async**: The helper function is `async fn` (not sync with `block_on`) because it is called from async test functions where `block_on` would cause a deadlock.
