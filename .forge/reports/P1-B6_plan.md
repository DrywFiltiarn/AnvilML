## Objective

Create the `anvilml-openapi` binary crate as a minimal stub that prints `"openapi generation stub"` and exits 0. Also create the `api/` directory with a `.gitkeep` placeholder so the `openapi-drift` CI gate has a real command to run from day one. Add the crate to the workspace `members` list. This establishes the build-time OpenAPI tooling scaffold before any real OpenAPI emission code is written in a later phase.

## Scope

### In Scope
- Create `crates/anvilml-openapi/Cargo.toml` as a binary crate with path dependencies on `anvilml-core` and `anvilml-server`.
- Create `crates/anvilml-openapi/src/main.rs` with a `fn main()` that prints `"openapi generation stub"` to stdout and exits 0.
- Create `api/.gitkeep` (empty file) to ensure the `api/` directory is tracked in git.

- Any test file — the acceptance criterion is a simple `cargo run` smoke test, not a separate test crate.
- Version bump — this task creates a new crate whose version is inherited from the workspace default (`0.1.0`); no source modification to an existing crate occurs, so §12 of ENVIRONMENT.md does not trigger a bump.

## Approach

1. **Create `crates/anvilml-openapi/Cargo.toml`.** Write a minimal Cargo manifest:
   - `[package]` block with `name = "anvilml-openapi"`, `version.workspace = true`, `edition.workspace = true`, `rust-version.workspace = true`.
   - `[dependencies]` section with `anvilml-core = { path = "../anvilml-core" }` and `anvilml-server = { path = "../anvilml-server" }`. These are path dependencies as specified by the task context; they are listed even though the stub binary does not call into them at compile time, because the task explicitly requires this dependency relationship.
   - No `[features]` section is needed — the `mock-hardware` feature forwarding pattern applies to crates that depend on `anvilml-hardware`, which this crate does not.

2. **Create `crates/anvilml-openapi/src/` directory.** Then create `crates/anvilml-openapi/src/main.rs` with:
   ```rust
   fn main() {
       println!("openapi generation stub");
   }
   ```
   This is a synchronous `fn main()` — no async, no tokio, no tracing. The real OpenAPI emission will need these when implemented, but the stub does not.

3. **Create `api/.gitkeep`.** Create the `api/` directory at the repository root if it does not exist, then create an empty file named `.gitkeep` inside it. This ensures `git` tracks the directory so the `openapi-drift` CI job has a real path to diff against once `api/openapi.json` is generated by a later task.

4. **Update root `Cargo.toml`.** Append `"crates/anvilml-openapi"` to the `members` array on line 3. The array currently contains 9 entries; this adds the 10th. Maintain the existing format (comma-separated quoted strings within square brackets).
