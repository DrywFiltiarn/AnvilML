## Summary

Fixed the venv path resolution in `anvilml-worker::spawn()` so that relative `venv_path` values are resolved against the repository root (via `_repo_root_for_worker()`) instead of `std::env::current_dir()`. This eliminates ENOENT / ERROR_PATH_NOT_FOUND failures when `cargo test` sets the process CWD to the crate directory rather than the repo root. The change is a single-line replacement in the `abs_venv` resolution block inside `spawn()`, plus a patch version bump from 0.1.8 to 0.1.9.

## Resolved Dependencies

No new dependencies added or modified. This task performs a targeted bug fix with zero dependency changes.
