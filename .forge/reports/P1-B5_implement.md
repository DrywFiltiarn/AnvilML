## Deviations from Plan

- **Added `anvilml-hardware` as a direct dependency** to both `anvilml-scheduler` and
  `anvilml-server` Cargo.toml files. The plan listed only the four transitive dependencies
  for scheduler and five for server. Cargo does not allow feature forwarding to transitive
  dependencies — the `[features] mock-hardware = ["anvilml-hardware/mock-hardware"]` entry
  requires `anvilml-hardware` to be a direct dependency of the crate declaring the feature.
  This is a minimal, non-breaking addition: `anvilml-hardware` is a zero-cost stub crate
  with no runtime code, so adding it as a direct dependency has no behavioral impact at
  this phase. This deviation is necessary for the feature forwarding to compile.
