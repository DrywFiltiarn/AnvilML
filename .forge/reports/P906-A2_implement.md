## Summary

Implemented three OpenAPI spec annotation fixes: added an inline schema to the artifact serve endpoint's 200 response, marked `id` and `job_id` as required in `ArtifactMeta`, and marked `fp32` as required in `InferenceCaps`. The `body = inline(schema!({...}))` syntax from the plan was not valid for utoipa 5.5.0 (the `schema!` macro expects a Rust type, not a JSON object), so `body = inline(String)` was used instead, producing `{"type": "string"}` in the generated OpenAPI spec. The `format: binary` field could not be injected through the `#[utoipa::path]` macro syntax because utoipa's `ComponentSchema` only adds format fields for specific built-in types (Uuid, DateTime). The `content_type = "image/png"` annotation already correctly identifies the response as PNG.
