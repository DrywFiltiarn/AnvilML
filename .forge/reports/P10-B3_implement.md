## Summary

Fixed a race condition in `ManagedWorker::spawn()` where `InitializeHardware` was written to the child process stdin pipe before `IpcHandles` (containing stdout) were delivered to `run_loop`. On Linux, `ChildStdout` uses edge-triggered epoll: data arriving before the fd is registered with epoll via the first `poll_read` is never signalled, so `reader_task` blocks forever waiting for a notification that will never come.

The fix ensures the reader task registers stdout with epoll *before* any data arrives by:
1. Delivering handles via oneshot immediately after stderr detach (before any writes)
2. Sending `InitializeHardware` through the mpsc channel instead of direct pipe writes
3. Yielding multiple times to allow writer_task and reader_task to complete before spawn() enters its status polling loop

Additionally, a pre-existing serialization format incompatibility was fixed: Python sends flat dicts with `_type` key (e.g., `{"_type": "Ready", ...}`) but Rust's rmp_serde named enum serialization produced nested maps (`{"Ready": [...]}`). The `write_frame` and `read_frame` functions in `anvilml-ipc` were updated to use a consistent flat dict format compatible with Python's msgpack serialization.

## Deviations from Plan

The plan specified using `self.tx.send(init_msg).await` (mpsc channel) for InitializeHardware delivery. This was implemented as planned, but required additional yields (3x `yield_now()`) to ensure writer_task and reader_task complete before spawn() enters its polling loop — a timing requirement not anticipated in the original plan.

Additionally, a pre-existing serialization format incompatibility between Rust (rmp_serde named enum) and Python (flat dict with `_type` key) was discovered and fixed. This involved:
- Adding `serialize_message()` function to produce flat dicts for WorkerMessage
- Updating `read_frame()` to deserialize flat dicts into WorkerEvent variants
- Updating all affected tests to use the new format

The plan also stated "No new tests are written (P10-B4 handles the regression test)" — no new tests were added, but existing tests were updated to use the correct serialization format.

A pre-existing test isolation bug in `handshake_completes_once` (env var save/restore) was fixed to prevent it from affecting subsequent tests when run together.
