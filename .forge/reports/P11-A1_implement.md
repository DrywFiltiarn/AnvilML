## Deviations from Plan

1. **`hashbrown` version:** The plan specified `hashbrown = "1"`. Since `hashbrown` has not reached a 1.x release (latest is 0.17.1), the constraint was written as `hashbrown = "0.17"` per the project's `major.minor` pinning convention.

2. **`update_from_worker` signature:** The plan's task description listed the signature as `update_from_worker(&self, types: Vec<NodeTypeDescriptor>)`, but ENVIRONMENT.md §9 mandates a `worker_id=` structured field in the DEBUG log. The ACT agent added `worker_id: &str` as the first parameter per the plan's own Deviations section. This matches the plan's Public API Surface table exactly.

3. **`PartialEq` not available:** `NodeTypeDescriptor` does not derive `PartialEq`, so the test `test_update_populates_registry` could not use `assert_eq!` to compare full structs. The test was adjusted to compare individual `type_name` fields instead.

4. **Borrow fix in `update_from_worker`:** The initial implementation used `for desc in types` which moved `types` before `types.len()` could be used in the debug log. Fixed by iterating over `&types` and cloning each descriptor during insertion.
