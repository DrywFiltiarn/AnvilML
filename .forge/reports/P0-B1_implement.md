## Deviations from Plan

- **`anvilml-scheduler` direct dependency on `anvilml-hardware`**: The plan listed `anvilml-scheduler` as forwarding `mock-hardware` from `anvilml-hardware`, but `anvilml-scheduler` does not directly depend on `anvilml-hardware` in the dependency graph (it depends on `anvilml-worker` which depends on `anvilml-hardware`). Cargo requires a direct dependency to forward a feature. Added `anvilml-hardware = { path = "../anvilml-hardware" }` to `anvilml-scheduler`'s `[dependencies]` to enable feature forwarding. This is a minimal change that preserves the dependency graph's logical flow while satisfying Cargo's feature resolution.
