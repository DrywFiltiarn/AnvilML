## Objective

Correct three secondary defects in the OpenAPI spec annotations so that `backend/openapi.json`
generated by `anvilml-openapi` will contain: an `image/png` binary schema on the
`GET /v1/artifacts/{hash}` 200 response, `id` and `job_id` in `ArtifactMeta.required`,
and `fp32` in `InferenceCaps.required`.

## Scope

### In Scope
- Add binary schema to `serve_artifact` 200 response `#[utoipa::path]` annotation
- Add `#[schema(required)]` to `ArtifactMeta.id` and `ArtifactMeta.job_id`
- Add `#[schema(required)]` to `InferenceCaps.fp32`

### Out of Scope
- Regenerating `backend/openapi.json` (owned by P906-A4)
- Bumping crate versions (P906-A1 bumps anvilml-openapi; P906-A3 bumps anvilml-core)
- Any handler logic changes
- Any new tests

## Approach

1. **Fix 1 — image/png binary schema:** In `crates/anvilml-server/src/handlers/artifacts.rs`,
   modify the `#[utoipa::path]` annotation on `serve_artifact` (line 38) from:
   ```rust
   (status = 200, description = "Artifact found", content_type = "image/png"),
   ```
   to:
   ```rust
   (status = 200, description = "Artifact found", content_type = "image/png",
    body = inline(schema!({"type": "string", "format": "binary"}))),
   ```
   This requires importing `utoipa::schema` if not already available. The `schema!` macro
   is provided by utoipa 5.x for inline schema definitions.

2. **Fix 2 — ArtifactMeta required fields:** In `crates/anvilml-core/src/types/artifact.rs`,
   add `#[schema(required)]` to the `id` and `job_id` fields (lines 16 and 21):
   ```rust
   #[serde(default)]
   #[schema(required)]
   pub id: Uuid,
   ...
   #[serde(default)]
   #[schema(required)]
   pub job_id: Uuid,
   ```

3. **Fix 3 — InferenceCaps required fields:** In `crates/anvilml-core/src/types/hardware.rs`,
   add `#[schema(required)]` to the `fp32` field (line 67):
   ```rust
   #[serde(default)]
   #[schema(required)]
   pub fp32: bool,
   ```

4. **Verify:** Run `cargo clippy -p anvilml-openapi -- -D warnings` — must exit 0.
   This single command exercises all three modified files through the dependency chain
   (`anvilml-openapi` → `anvilml-server` → `anvilml-core`).
