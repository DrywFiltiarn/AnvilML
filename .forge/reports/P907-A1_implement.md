## Summary

Added `zeromq = { version = "0.4", features = ["tokio"] }` to the workspace dependencies in root `Cargo.toml` and to `crates/anvilml-worker/Cargo.toml`. Removed the `interprocess` dependency from `anvilml-worker` and migrated the IPC layer from `interprocess::local_socket` to tokio's `TcpListener`/`TcpStream`, which provides cross-platform compatibility (Linux + Windows). The Python worker's `ipc.py` was updated to detect TCP addresses and connect via `AF_INET` instead of always using `AF_UNIX`. The `anvilml-worker` package version was bumped from `0.1.22` to `0.1.23`.

## Deviations from Plan

- The plan only specified Cargo.toml changes (add zeromq, remove interprocess). However, removing `interprocess` from dependencies while the source code still used it caused compilation failures. Per FORGE_AGENT_RULES §9.2, build failures caused by this task must be fixed. The source code in `crates/anvilml-worker/src/managed.rs` was migrated from `interprocess::local_socket` to `tokio::net::TcpListener`/`TcpStream` for cross-platform compatibility.
- The Python worker's `worker/ipc.py` was updated to support TCP address detection in the `connect()` function, since the IPC transport changed from Unix sockets to TCP.
- The IPC transport changed from filesystem-based Unix sockets/named pipes to TCP (`127.0.0.1:PORT`). This is consistent with the project's ZeroMQ TCP-based IPC architecture described in ENVIRONMENT.md §3.7 (`ANVILML_IPC_PORT`).
