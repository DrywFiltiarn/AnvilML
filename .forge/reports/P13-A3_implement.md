## Summary

Implemented the dispatch loop for `JobScheduler` (P13-A3). The loop uses `Arc<Notify>` for
wake-up and `tokio::time::timeout(100ms, notify.notified())` for reliable polling. On each
wake it pops the next queued job, selects an idle worker via `select_worker()`, transitions
the job to Running, marks the worker busy, broadcasts `JobStarted`, and sends `Execute` to
the worker. This repeats until no more jobs can be dispatched, then waits on Notify.
Additionally, the `JobScheduler::new()` signature was updated to match the new constructor
arguments (replacing `Arc<Vec<WorkerInfo>>` and `Arc<Notify>` with `Arc<WorkerPool>` and
`Arc<tokio::sync::Mutex<VramLedger>>` plus `default_device`), requiring updates to
`backend/src/main.rs` and the test helper in `anvilml-server/src/handlers/jobs.rs`.
10 new unit tests were added covering dispatch loop behavior and select_worker variants.
The `clippy::while_let_loop` lint was fixed by rewriting the dispatch loop body as a
`while let` instead of `loop { if break }`.
