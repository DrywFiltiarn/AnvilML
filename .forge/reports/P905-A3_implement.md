## Summary

Added safetensors header-based dtype detection to the model registry scanner. The new `read_safetensors_dtype` function reads the 8-byte LE u64 header length, parses the JSON header, counts dtype occurrences across tensor keys (excluding `__metadata__`), and returns the most-frequent dtype mapped to `DType`. The `scan_dirs` function now calls this for `.safetensors` files before falling back to filename-based `infer_dtype`. Added 8 unit tests and 1 integration test. Bumped `anvilml-registry` patch version from 0.1.1 to 0.1.2.

## Resolved Dependencies

No new dependencies were added. All functionality uses existing dependencies (`std::fs::File`, `std::io::Read`, `serde_json`, `tempfile` dev-dependency).
