## 1. What AnvilML Is

AnvilML is the Rust backend binary (`anvilml`) of the SindriStudio image-generation
platform. It:

- Spawns and supervises one Python worker process per GPU (or one CPU worker as fallback).
- Exposes a versioned REST + WebSocket API as the sole integration surface for any frontend.
- Manages job scheduling, the model registry, artifact storage, and all persistent state.
- Optionally serves a frontend (local files, reverse-proxy, or headless API-only mode).

BloomeryUI is the reference frontend. The frontend is architecturally interchangeable;
AnvilML never embeds frontend assets.

## 3. Crate Dependency Graph

```
anvilml-core  (no deps)
  ├── anvilml-hardware        (← core)
  ├── anvilml-registry        (← core)
  ├── anvilml-ipc             (← core)
  └── anvilml-worker          (← ipc, hardware, core)
        └── anvilml-scheduler (← worker, registry, core)
              └── anvilml-server  (← all above)
                    └── backend/src/main.rs  (← server)

anvilml-openapi  (← core, server — build-time dev binary only, not in release binary)
```

**Rule:** every crate depends only on crates below it in this graph. No cycles. Any
new dependency must follow this order.

## 4. Crate Responsibilities

| Crate | Responsibility | Key modules |
|-------|----------------|-------------|
| `anvilml-core` | Pure data: domain types, config, errors. Zero I/O, zero async. | `config.rs`, `error.rs`, `types/{job,model,artifact,hardware,worker,events}.rs` |
| `anvilml-hardware` | Detect GPUs/CPU; refreshable VRAM snapshot. | `lib.rs` (`DeviceDetector` trait, `detect_all_devices`), `cuda.rs`, `rocm.rs` (Linux + Windows), `cpu.rs`, `mock.rs` |
| `anvilml-registry` | Scan model dirs, persist `ModelMeta` to SQLite, serve queries. | `scanner.rs`, `store.rs`, `lib.rs` |
| `anvilml-ipc` | IPC message enums + length-prefixed msgpack framing. | `messages.rs`, `framing.rs` |
| `anvilml-worker` | Spawn/supervise/respawn workers; IPC bridge; env injection. | `pool.rs`, `managed.rs` (spawn + stdin/stdout IPC bridge), `env.rs` |
| `anvilml-scheduler` | Job queue, VRAM ledger, DAG validation, dispatch loop. | `queue.rs`, `ledger.rs`, `dag.rs`, `nodes.rs` (`KNOWN_NODE_TYPES`), `job_store.rs`, `scheduler.rs` |
| `anvilml-server` | axum router, handlers, WS broadcaster, artifact store, frontend serving. | `lib.rs`, `state.rs`, `handlers/*.rs`, `ws/{broadcaster,handler,stats_tick}.rs`, `artifact/store.rs`, `frontend.rs` |
| `anvilml-openapi` | Emit `openapi.json` from utoipa annotations (build-time only). | `main.rs` |

## 5. Cargo Feature Flags

| Feature | Crates | Effect |
|---------|--------|--------|
| `mock-hardware` | `anvilml-hardware` (forwarded by `worker`, `scheduler`, `server`) | Replaces real GPU detection with a deterministic stub driven by `ANVILML_MOCK_*` env vars. **All CI builds use this flag.** |

**Forwarding rule:** every crate that depends on `anvilml-hardware` must forward the
feature in its `Cargo.toml`:
```toml
[features]
mock-hardware = ["anvilml-hardware/mock-hardware"]
```

The Python-side mock (`ANVILML_WORKER_MOCK=1`) is a separate, orthogonal mechanism.

## 6. IPC Protocol Summary

**Transport:** child process stdin/stdout pipes (not TCP/UDS).

**Framing:** `[ 4 bytes big-endian u32: payload_len ] [ payload_len bytes: msgpack ]`

**Max payload:** `limits.max_ipc_payload_mib` (default 64 MiB). Exceeding the limit or
a deserialisation failure causes immediate worker kill + respawn.

**Windows requirement:** `worker/ipc.py` must set stdin/stdout to binary mode at import
time using `msvcrt.setmode`. This guard must exist from the first commit. See
`ANVILML_DESIGN.md §7.1`.

**Message directions:**
- Rust → Python (`WorkerMessage`): `Ping`, `Shutdown`, `InitializeHardware`, `Execute`, `CancelJob`, `MemoryQuery`
- Python → Rust (`WorkerEvent`): `Pong`, `Ready`, `Progress`, `ImageReady`, `MemoryReport`, `Completed`, `Failed`, `Cancelled`, `Dying`

Full field specifications: `ANVILML_DESIGN.md §7.2–7.3`.

## 7. HTTP API Surface (summary)

Base path: `/v1/`

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/health` | Health check: `{ status, version, uptime_s }` |
| `GET` | `/v1/system` | Hardware info (`HardwareInfo`) |
| `GET` | `/v1/system/env` | Python env preflight report (`EnvReport`) |
| `POST` | `/v1/jobs` | Submit a new job (validates graph; 422 on invalid, 202 on accept) |
| `GET` | `/v1/jobs` | List jobs (`?status=`, `?limit=`, `?before=`) |
| `GET` | `/v1/jobs/:id` | Get one job |
| `POST` | `/v1/jobs/:id/cancel` | Cancel a queued or running job |
| `DELETE` | `/v1/jobs/:id` | Delete one terminal job + its artifacts |
| `DELETE` | `/v1/jobs` | Bulk clear (`?status=completed\|failed\|cancelled\|all`) |
| `GET` | `/v1/models` | List scanned models (`?kind=`) |
| `GET` | `/v1/models/:id` | Get one model |
| `POST` | `/v1/models/rescan` | Trigger model directory rescan |
| `GET` | `/v1/workers` | List worker pool status |
| `POST` | `/v1/workers/:id/restart` | Restart a specific worker |
| `GET` | `/v1/artifacts` | List artifacts (`?job_id=`) |
| `GET` | `/v1/artifacts/:hash` | Fetch artifact PNG bytes |
| `GET (WS)` | `/v1/events` | WebSocket event stream (deltas only; no history replay) |

Full request/response shapes: `ANVILML_DESIGN.md §10`.

**WebSocket events** (`WsEvent`): `job.queued`, `job.started`, `job.progress`,
`job.image_ready`, `job.completed`, `job.failed`, `job.cancelled`, `worker.status`,
`system.stats`. Serialised as `{ "event": "<name>", "timestamp": "<ISO8601>", ...fields }`.
Full enum: `ANVILML_DESIGN.md §4.5`.

## 8. SQLite Schema (summary)

Three tables, managed by sqlx migrations in `backend/migrations/`:

- `jobs` — job lifecycle state, graph JSON, settings, timestamps
- `models` — scanned model metadata (path, kind, dtype, hash, file size)
- `artifacts` — generated image metadata (job_id FK, path, hash, dimensions)

Full DDL: `ANVILML_DESIGN.md §13`.

## 9. CI Matrix

| Job | Runner | Commands |
|-----|--------|----------|
| `rust` | `ubuntu-latest` | `cargo fmt --all --check`, `cargo clippy … --features mock-hardware -D warnings`, `cargo test … --features mock-hardware` |
| `python-worker` | `ubuntu-latest` | `pip install -r worker/requirements/base.txt`, `ANVILML_WORKER_MOCK=1 python -m pytest worker/tests/ -v` |
| `openapi-diff` | `ubuntu-latest` | `cargo run -p anvilml-openapi`, `git diff --exit-code backend/openapi.json` |
| `rust-windows` | `windows-latest` | `cargo clippy … --features mock-hardware -D warnings`, `cargo test … --features mock-hardware` |

`python-worker` and `openapi-diff` declare `needs: [rust]`. `rust-windows` runs in
parallel with the Linux jobs. The base CI (Linux fmt+clippy+test, Windows clippy+test)
is established in `TASKS_PHASE001.md §P1-A5`; the `openapi-diff` gate and `python-worker`
job are added in `TASKS_PHASE020.md §P20-A4`.

> CI runners have no GPU, so all jobs build with `mock-hardware` / `ANVILML_WORKER_MOCK=1`.
> Real CUDA and ROCm execution — including **ROCm on Windows** — is validated by the manual
> smoke tests in `ANVILML_DESIGN.md §20.4`, not by CI.

## 10. Platform Notes

Linux and Windows are co-equal first-class targets (`ANVILML_DESIGN.md §1.5`).

- **Binary-stdio guard** in `worker/ipc.py` must be present from the first commit.
- **`.gitattributes`** locks line endings: `*.sh eol=lf`, `*.ps1 eol=crlf`, `*.py eol=lf`, `*.rs eol=lf`.
- **`PR_SET_PDEATHSIG`** (Linux) / **Job Object** (Windows) used to clean up orphaned workers on hard kill.
- **ROCm on Windows is a mandatory MVP backend** (`ANVILML_DESIGN.md §5`): AMD's *PyTorch on Windows* package (ROCm ≥ 7.2) on supported Radeon RX 7000/9000-series + select Ryzen AI hardware; detection falls back to a HIP torch-probe where the Linux CLIs are absent. DirectML is the only deferred AMD path (§25).

## 11. Key Design Decisions (for context, not re-litigated)

- **No embedded frontend assets.** BloomeryUI is distributed and served separately.
- **One Python worker per device.** Avoids GPU memory fragmentation from multi-process inference.
- **ROCm is a first-class MVP backend on both Linux and Windows** — Linux via the pip ROCm wheels, Windows via AMD's *PyTorch on Windows* package (ROCm ≥ 7.2). DirectML is deferred (only for AMD GPUs outside ROCm-on-Windows support).
- **stdin/stdout IPC, not TCP.** Avoids port allocation and per-platform socket handling.
- **`anvilml-core` has zero I/O and zero async.** Keeps domain types testable without a runtime.
- **`mock-hardware` feature flag**, not runtime env, controls the detection path in Rust. This ensures the real detectors are always compiled and linted, even in CI.
- **The Forge commits.** Cline never calls `git commit` or `git push`. See `.clinerules §5`.
